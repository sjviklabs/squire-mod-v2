# Plan 05-04: Item Pickup with Junk Filtering

## Objective

Implement the ItemHandler behavior that makes the squire pick up nearby dropped ItemEntity instances, filtered against a configurable junk list. After this plan, items on the ground near the squire automatically enter its inventory (if space exists and the item isn't junk).

<action>
Create `src/main/java/com/sjviklabs/squire/ai/handler/ItemHandler.java`:

1. **Scan logic**: Every 10 ticks, scan for `ItemEntity` instances within `SquireConfig.PICKUP_RANGE` (default 3.0 blocks) using `level.getEntitiesOfClass(ItemEntity.class, squire.getBoundingBox().inflate(range))`.

2. **Junk filter**: Read `SquireConfig.JUNK_FILTER_LIST` — a `List<String>` of resource locations (e.g., `"minecraft:cobblestone"`, `"minecraft:dirt"`). Parse each to `ResourceLocation`, compare against `item.getItem().getItem()` registry name via `BuiltInRegistries.ITEM.getKey()`. Skip items on the list.

3. **Insertion**: For each valid item, call `squireItemHandler.insertItem(slot, stack, false)` iterating slots. If insertItem returns non-empty (partial insert), update the ItemEntity's stack. If fully inserted, call `itemEntity.discard()`.

4. **Capacity check**: Skip pickup entirely if all accessible slots return non-empty from `insertItem(slot, stack, true)` (simulate mode).

5. **FSM integration**: Register as PICKING_UP_ITEM state in SquireBrain. Priority 45 (work tier). Transition condition: idle or following AND items nearby AND inventory has space.

Add to `SquireConfig.java` in the `[inventory]` section:
- `PICKUP_RANGE = defineInRange("pickupRange", 3.0, 1.0, 8.0)`
- `JUNK_FILTER_LIST = defineList("junkFilter", List.of("minecraft:cobblestone", "minecraft:dirt", "minecraft:gravel", "minecraft:rotten_flesh", "minecraft:poisonous_potato"), () -> "", s -> s instanceof String)`

Register `ItemHandler` in `SquireBrain` constructor alongside other handlers.
</action>
