WHY horse.move(MoverType.SELF) is mandatory:
- AbstractHorse.getControllingPassenger() → instanceof Player hardcoded at line 1081
- LivingEntity.aiStep() gates travelRidden() on getControllingPassenger() instanceof Player (line ~2816)
- When squire (PathfinderMob) is the passenger, vanilla travel() receives zero input — horse sits still
- horse.getNavigation().moveTo() also has no effect when the horse has a passenger
- ONLY horse.move(MoverType.SELF, vec) bypasses this — confirmed working in v0.5.0

driveHorseToward() MUST include gravity:
```java
if (!horse.onGround()) {
    horse.move(MoverType.SELF, new Vec3(0, -0.08, 0));  // explicit gravity — vanilla travel() bypassed
}
```

driveHorseToward() MUST include wall step-up:
```java
if (horse.horizontalCollision && horse.onGround()) {
    horse.move(MoverType.SELF, new Vec3(0, 0.6, 0));
    horse.move(MoverType.SELF, movement);  // retry after step-up
}
```
