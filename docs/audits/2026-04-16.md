## Medium findings (tech debt — in GTD inbox)

Architecture:
- **`SquireBrain.dispatch()` still has 7-case switch on `task.commandName`.** The v3.0.0 refactor claimed "WorkHandler replaced switch-statement dispatch" but this path was missed. Task dispatch should follow the same polymorphic pattern as work suspension.
- **`TorchHandler` ticked as side-effect in `FollowHandler` transition** (`SquireBrain.java:707`) rather than via WorkHandler registry. Breaks the claimed "one line in `workHandlers`" pattern. Either add as WorkHandler or document why it's excluded.
- **`DangerHandler` is a 49-LOC shallow module** (Ousterhout Ch. 4). Single public method delegating to CombatHandler. Merge into CombatHandler unless expansion is planned.

Code quality:
- **6 handlers 500+ LOC**: MiningHandler (883), CommandHandler (992 — note: may be SquireCommand), CombatHandler (761), FarmingHandler (710), FishingHandler (511), ChestHandler (539). Extract strategy classes: e.g. `MiningBreakStrategy`, `MiningApproachStrategy`.
- **`SquireCommand.onRegisterCommands()` is a 192-line single method.** Brigadier tree is readable but brittle. Split by subcommand group.
- **NBT save/load has no version byte.** Old saves with missing fields silently default. Add a `SaveVersion` byte + migration fallback.
- **`/squire name <text>` greedyString has no length limit.** Add 32-char cap to prevent chat-formatting abuse.

Supply chain:
- **No `gradle.lock`.** Transitive dependencies unpinned. Run `./gradlew dependencies --write-locks`, commit the output.
- **Custom Maven repos** (cloudsmith.io, cursemaven.com, jfrog.io, modrinth.com) are unvetted relative to Maven Central. Audit integrity.
