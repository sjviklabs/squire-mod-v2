# Squire — Design Handoff

**Package contents for the v3.0.1 distribution push.**

You landed on **Seal (A)** as the primary logo, **Brass** as the accent, **Banner** as the hero wordmark style. Everything here is built around those choices.

## Files in this folder

| File | What it is | What to do with it |
|---|---|---|
| `../Squire Design.html` | Live design file — open in a browser | Reference for all visuals; use Tweaks panel to explore |
| `BRAND.md` | Visual identity summary (colors, type, motifs) | Paste into a wiki or keep as the source of truth |
| `LISTING_COPY.md` | Paste-ready Modrinth / CurseForge copy | Drop into the description field on each platform |
| `SCREENSHOTS.md` | In-game capture guide | Hand to whoever's taking the shots (or follow yourself) |
| `LOGO_EXPORTS.md` | Export checklist for the chosen Seal mark | Work list for producing final SVG/PNG assets |
| `NEXT_ROUND.md` | What to brief Claude for round 2 | Prompts for landing page + in-game UI mocks |

## Order of operations

1. **Read `BRAND.md`** — 5 min, sets the visual rules.
2. **Capture screenshots** per `SCREENSHOTS.md` — this is the unlock; listing can't ship without them.
3. **Export the logo** per `LOGO_EXPORTS.md` — produce SVG + PNG at the sizes Modrinth/CurseForge need.
4. **Paste `LISTING_COPY.md`** into Modrinth, then CurseForge. Upload the logo as the icon, screenshots in the gallery order specified.
5. **Ship**, then brief round 2 per `NEXT_ROUND.md`.

## One-line summary

You have a functional mod with zero distribution. This package is the minimum-viable set of assets to change that — brand, copy, and a capture plan that turns 20 minutes of in-game time into a shippable listing on both storefronts.
